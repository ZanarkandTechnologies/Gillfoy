# Commit Message Maintenance

## Scope

- `SKILL.md`
- `references/style.md`
- `README.md`

## Boundaries

- Keep this skill about commit subjects, not full commit bodies.
- Keep rules compact and repo-specific.
- Prefer one canonical style shape.

## Conventions

- Lead with `Best:`.
- Keep the default output tiny.
- Reference `MEM-0009` for the commit-subject rule.

## Checks

- Trigger conditions, workflow, gotchas, and outcome contract exist.
- `type(scope): summary` remains the default shape.
- Examples match observed repo style.

## Testing

- Re-read `SKILL.md` once and confirm it works without opening references.
- Confirm the reference only adds type/scope hints, not a second conflicting style.
