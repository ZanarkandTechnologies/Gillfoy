# Understand First

Use when the codepath is unfamiliar and fixing immediately would be guesswork.

## Strategy

1. Explain the system in simple terms before proposing fixes.
2. Identify responsibilities, dependencies, callers, side-effects, and state transitions.
3. Call out the parts most likely to explain the symptom.
4. Only then move into runtime-debugging or direct fixing.

## Output

- system overview
- key dependencies and callers
- state or data flow summary
- likely risk points
