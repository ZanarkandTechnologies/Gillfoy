# Debugging Knowledge Base

Use when a solved bug should become reusable team context.

## Capture

- symptom
- trigger conditions
- root cause
- useful commands or dashboards
- known flaky tests
- workaround and permanent fix

## Where To Store

- repo debugging rules or `.cursor/rules/*`
- `docs/MEMORY.md` for durable constraints
- `docs/HISTORY.md` for append-only change history

## Rule

If the same class of bug is likely to recur, write it down before closing the task.
